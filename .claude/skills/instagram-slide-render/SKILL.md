# Instagram スライド画像レンダリングスキル

AI生成写真にテキストを重ね、投稿用の完成版PNGを出力する。

## 入力

以下の情報を受け取る:

1. **投稿フォルダパス**: 例 `marketing/sns/posts/post-01-hokkaido`
2. **S1写真ファイル名**: AI生成したメロン畑等の写真
3. **S3写真ファイル名**: AI生成したビールグラス等の写真
4. **テキスト内容**: `/instagram-post-create` の出力（各スライドのテキストとpx座標）

## 出力

4枚の完成版PNG:
- `slide-01-final.png` — S1写真 + テキスト + グラデーション暗幕
- `slide-02-final.png` — 黒背景 + EDUCATE テキスト
- `slide-03-final.png` — S3写真 + テキスト + グラデーション暗幕
- `slide-04-final.png` — 黒背景 + BRAND テキスト + ROOTSロゴ

## レンダリングルール

### 共通
- キャンバス: 1080 x 1350px
- フォント: `brand/assets/fonts/` から読み込み
  - ShipporiMincho-Regular.ttf
  - BebasNeue-Regular.ttf
  - SpaceMono-Regular.ttf
- カラー: CLAUDE.md準拠（bg=#090909, text=#e0e0e0, orange=#F0A032, blue=#50A0F0）

### S1・S3（写真スライド）のテキスト処理

**絶対にやること:**
1. 写真を 1080x1350 にリサイズ（LANCZOS）
2. テキスト領域にグラデーション暗幕を敷く（テキストの上下200px範囲）
3. 暗幕の上にテキストを描画（影なし。暗幕が視認性を確保する）

**グラデーション暗幕の仕様:**
```python
def add_gradient_band(img, y_start, y_end, opacity=200):
    overlay = Image.new('RGBA', (W, H), (0, 0, 0, 0))
    draw_o = ImageDraw.Draw(overlay)
    band_height = y_end - y_start
    for i in range(band_height):
        t = i / band_height
        alpha = int(opacity * (1 - (2 * t - 1) ** 2))
        draw_o.rectangle([0, y_start + i, W, y_start + i + 1], fill=(0, 0, 0, alpha))
    return Image.alpha_composite(img.convert('RGBA'), overlay).convert('RGB')
```

**絶対にやらないこと:**
- ❌ テキスト影（全方向に黒文字を重ねる方法） → 安っぽくなる
- ❌ テキストを写真の主役（食材・ビール）に重ねる → 主役を邪魔する
- ❌ テキストを多く載せすぎる → 写真スライドは最小限のテキスト

### S1のテキスト配置ルール
- メインテキスト: **写真の上部1/3** に配置（y=300〜450付近）
- 食材が画面中央〜下部にある場合、テキストは上部に
- 風景が画面上部にある場合、テキストは下部に（写真ごとに判断）
- サブテキスト（FLAVOR BLUEPRINT vol.XX）: y=1270付近、20px、控えめ

### S3のテキスト配置ルール
- テキスト: **写真の下部1/3** に配置（y=900〜1100付近）
- 3段構成:
  1. ラベル「相性の良いビアスタイル」（24px, #e0e0e0, 控えめ）
  2. スタイル名（Bebas Neue, 80px, #F0A032, 主役）
  3. 説明1行（36px, #e0e0e0）
- ビールグラスの上部は絶対にテキストで隠さない

### S2のレンダリング
- 黒背景(#090909)
- 左に装飾線（#50A0F0, 2px幅）
- 見出し + 4項目（AROMA/BITTER/SOUR/SWEET）
- テンプレートのpx座標に従う

### S4のレンダリング
- 黒背景(#090909)
- メッセージ（Shippori Mincho 52px）
- ROOTSロゴ（Bebas Neue 112px, #F0A032）
- アカウント名（Space Mono 24px, 50%透過, 右下）

## 実行手順

1. 投稿フォルダ内のAI写真を検索して特定する
2. S1の写真を読み込み、食材/風景の位置を確認し、テキスト位置を決定
3. S1のグラデーション暗幕 + テキストを描画 → PNG保存
4. S2の黒背景テキストを描画 → PNG保存
5. S3の写真を読み込み、テキストを下部に描画 → PNG保存
6. S4の黒背景 + ロゴを描画 → PNG保存
7. 生成した4枚を目視確認（Read toolで画像を表示）
8. 問題があれば修正して再生成

## 品質チェック（生成後に必ず実行）

生成した4枚すべてをRead toolで読み込み、プロのデザイナーの視点でチェックする。
さらに**スマホプレビュー（375px幅にリサイズ）**を生成し、実機での見え方を確認する。

### レベル1: 致命的エラー（1つでもあれば即修正）
- [ ] テキストが写真の主役（食材/ビール）に被っていないか
- [ ] テキストが読めないほど小さい・薄い箇所はないか
- [ ] 安っぽいテキスト影（にじみ、黒い縁取り）が出ていないか
- [ ] テキストの切れ・はみ出しがないか（セーフエリア80px内に収まっているか）
- [ ] ハードコードした数値や事実に誤りがないか

### レベル2: スマホ視認性（375px幅プレビューで確認）
- [ ] **全テキストが375px幅で読めるか**（最小フォント: 元画像で32px以上 = スマホで約11px）
- [ ] サブテキスト（FLAVOR BLUEPRINT等）がスマホで潰れていないか
- [ ] グラデーション暗幕がスマホサイズでも効いているか（テキストと背景のコントラスト）

### レベル3: コンポジション（構図・バランス）
- [ ] **写真の主役とテキストの位置関係**: 主役が活きているか。テキストは脇役になっているか
- [ ] **余白のバランス**: テキスト周辺に十分な余白があるか。詰まっていないか
- [ ] **視線の流れ**: S1→S2→S3→S4をスワイプしたとき、テキスト位置が上→左→下→中央と変化し、単調にならないか
- [ ] **テキストの密度**: 1スライドに情報を詰め込みすぎていないか。写真スライド（S1/S3）は最小限のテキスト

### レベル4: タイポグラフィ
- [ ] **行間**: フォントサイズの1.5倍が確保されているか（64px→行間96px）
- [ ] **文字間**: テキストが窮屈に見えないか
- [ ] **改行位置**: 単語の途中で改行されていないか。句点・読点で改行しているか
- [ ] **フォントの統一**: Shippori Mincho / Bebas Neue / Space Monoの3種のみ使用しているか

### レベル5: ブランド一貫性
- [ ] **カラー**: #e0e0e0（テキスト）, #F0A032（オレンジ）, #50A0F0（ブルー）, #090909（背景）のみ使用
- [ ] **S2/S4の黒背景**: 正確に#090909か（真っ黒#000000ではないか）
- [ ] **ROOTSロゴ**: Bebas Neue / #F0A032 / 112px。他のフォントやサイズになっていないか
- [ ] **フィード全体の一貫性**: 前の投稿（post-00-intro等）とトーンが揃っているか

### レベル6: Instagram固有の考慮
- [ ] **1枚目のサムネイル**: フィード3列グリッドで表示される正方形（上下がクロップされる）で、テキストが切れないか
- [ ] **テキスト位置のInstagram UI回避**: 下部にInstagramのUI要素（いいね・コメント等）が重なる領域を避けているか。y=1270以下にはテキストを置かない

### 過去の失敗パターン（再発防止）
- ❌ テキスト影を全方向に黒文字を重ねて実装 → 安っぽくなった → **グラデーション暗幕に変更**
- ❌ テキストが食材の上に被った → **テキストは上部1/3 or 下部1/3に配置**
- ❌ S3に情報を詰め込みすぎた → **写真スライドはテキスト最小限**
- ❌ サブテキストが小さすぎてスマホで読めなかった → **最小32px（元画像基準）**

### 品質チェックの実行方法

```
1. 4枚をRead toolで表示して目視確認
2. スマホプレビュー（375px幅）を生成して確認
3. 問題があればコードを修正して再生成
4. 再生成後、再度Read toolで確認
5. 全レベルクリアで完了
```

## プロレベル運用基準

このスキルはROOTSのブランドビジュアルを直接生成する。
**品質が低い画像は投稿しない。** 妥協するくらいなら再生成する。

### 判断原則

1. **Canvaで作ったほうがマシに見えるなら失敗**: Pillowの出力がCanva以下なら、方法を変える
2. **スマホで確認**: 必ず375px幅プレビューを生成する。PCでは良く見えてもスマホで破綻することがある
3. **写真の主役を殺さない**: テキストは写真の引き立て役。写真が見えなくなったら本末転倒
4. **暗幕は必須**: 写真の上にテキストを直接置かない。必ずグラデーション暗幕を敷く

### 数値基準（曖昧な判断を排除）

| 項目 | 基準 |
|------|------|
| 最小フォントサイズ | 32px（元画像基準）。これ以下はスマホで読めない |
| グラデーション暗幕の不透明度 | 200〜220。薄すぎるとテキストが読めない |
| テキストのセーフエリア | 上下左右80px以内にテキストを置かない |
| S1テキスト位置 | y=250〜500（上部1/3）または y=900〜1100（下部1/3） |
| S3テキスト位置 | y=880〜1100（下部1/3） |
| S1/S3のテキスト行数 | 最大3行。4行以上は情報過多 |

### エッジケース対応

| 状況 | 対処 |
|------|------|
| AI写真のアスペクト比が4:5でない | リサイズ時にクロップが必要。食材/ビールが切れないように調整 |
| 写真の上部が明るい（空など） | テキストを下部に配置。上部暗幕は効きにくい |
| 写真の下部が明るい（木のテーブルなど） | 暗幕のopacityを220→240に上げる |
| テキストが長くて1行に収まらない | 強制改行するのではなく、テキスト自体を短くする（post-createスキルに差し戻す） |

### 過去の失敗ログ

1. **テキスト影事件**: 全方向に黒文字を重ねて影を作った → 安っぽくなった → グラデーション暗幕に変更
2. **メロン被り事件**: テキストがメロンの真上に配置された → テキストを上部1/3に移動
3. **S3情報過多事件**: S3に「サブ見出し+スタイル名+説明2行」を詰め込んだ → スタイル名+説明1行に削減
4. **サブテキスト小さすぎ事件**: 24pxのラベルがスマホで読めなかった → 最小32pxに変更
