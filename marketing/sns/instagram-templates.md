# ROOTS Instagram 投稿テンプレート（確定版）

投稿制作に必要な素材を一括で生成するためのテンプレート集。
スキル `/instagram-post-create` がこのファイルを参照して投稿素材を出力する。

---

## 使い方

```
/instagram-post-create
県: 北海道、食材: メロン、型: 情景型
```

---

## 共通ルール

### キャンバス
- サイズ: **1080 x 1350px**（4:5）
- 背景: #090909
- セーフエリア: 上下左右 **80px** の余白を確保（テキストはこの内側に収める）
- テキストエリア: x=80〜1000, y=80〜1270（有効幅920px, 有効高1190px）

### カラー
- 背景: #090909
- テキスト: #e0e0e0
- アクセント暖色: #F0A032（オレンジ）
- アクセント寒色: #50A0F0（ブルー）

### フォント（実寸px / 画像書き出し時のサイズ）
| 用途 | フォント | サイズ | 色 |
|------|---------|-------|-----|
| S1メインテキスト | Shippori Mincho | 64px | #e0e0e0 |
| S1サブテキスト | Space Mono | 28px | #50A0F0 / 60%透過 |
| S2見出し | Shippori Mincho | 56px | #e0e0e0 |
| S2ラベル | Space Mono | 28px | #F0A032 |
| S2本文 | Shippori Mincho | 36px | #e0e0e0 |
| S3スタイル名 | Bebas Neue | 64px | #F0A032 |
| S3説明文 | Shippori Mincho | 36px | #e0e0e0 |
| S4メッセージ | Shippori Mincho | 52px | #e0e0e0 |
| S4ロゴ | Bebas Neue | 112px | #F0A032 |
| S4アカウント名 | Space Mono | 24px | #e0e0e0 / 50%透過 |

### 画像テキスト文字数（厳守）
| スライド | 理想 | 上限 |
|---------|------|------|
| 1（HOOK） | 10〜20文字 | 30文字 |
| 2（EDUCATE） | 40〜70文字 | 100文字 |
| 3（IMAGINE） | 30〜50文字 | 75文字 |
| 4（BRAND） | 8〜15文字+ロゴ | 30文字 |

上限を超える内容はキャプション（投稿文）に移す。

### 改行ルール
- 句点「。」の後で改行する
- 読点「、」の後でも改行してよい（長い場合）
- 1行あたり最大 **15文字** を目安にする（64pxの場合）
- 行間: フォントサイズの **1.5倍**（64px → 行間96px）

### ビジュアルのリズム
```
S1: AI生成写真 + テキスト（止める）
S2: 黒背景 + テキスト図解（読ませる）
S3: AI生成写真 + テキスト（欲しがらせる）
S4: 黒背景 + ロゴ（覚えさせる）
```

### 投稿文
- tone_guide.md準拠
- 敬語ベース、体言止め可
- 禁止: お得・限定・ゲット・超・やばい・神・!!
- 主語は「あなた」「この味」「ルーツ」。ROOTSを主語にしない
- 「○○の人なら知っている」のようなよいしょ表現は禁止
- 「[県名]を一杯のビールにするなら」の視点をキャプションに自然に入れる
- 既存ビールとの差別化はキャプション内で行う（画像には入れない）
- **食材を断定しない**: 投票で決めるのがROOTSのコンセプト。「たとえば」「もし」「あなたなら」等を使い、食材が確定していないことを示す
  - ✅ 「たとえば、富良野のメロン。」「もしこの味がビールになったら。」
  - ❌ 「北海道のビールはメロンで作ります。」「メロンに決まりました。」
- **設計図は"案のひとつ"**: 「これが、設計図。」ではなく「これが、設計図のひとつ。」
- **CTAで選択を促す**: 「あなたなら、どの味を選びますか。」のように、読者の主体性を尊重する

## 生成フロー（1投稿分）

`/instagram-post-create` スキルが以下を自動実行する:

1. **テーマ決定**: 県 + 食材 + 型（情景/記憶/事実/対比/体感/声）を受け取る
2. **Phase 1 ワード選定**: ハッシュタグ15〜20個を選定
3. **Phase 2 コピーライティング**: S1〜S4テキスト + キャプション生成（改行位置明示）
4. **Phase 3 ビジュアルディレクション**: 画像プロンプト + px座標テキスト配置を出力
5. **レビュー**: tone_guide.mdとの整合性・文字数制限をチェック
